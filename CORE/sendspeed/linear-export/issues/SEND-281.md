## Descrição

> **Como** PO
> **Quero** desenvolver um novo smartblock utilizando como base um smartblock que já foi testado e validado
> **Para** acelerar o desenvolvimento da consultoria e escalar uma solução que já demonstrou bons resultados

---

## Critérios de aceite:

* O novo Smartblock deve partir de um modelo já testado anteriormente
* Deve ser possivel criar um Smartblock através da nossa plataforma.
* O Smartblock deve conseguir se encaixar facilmente na <div> selecionada no lugar do editors.
* O comportamento esperado do Smartblock deve manter os padrões desenvolvidos no teste
* Deve ser possível adaptar o Smartblock para novos contextos ou campanhas, sem perder sua essência.
* A alteração da <div> do smartblock deve ser realizada de maneira instantânea.
* O desenvolvimento deve priorizar agilidade e reaproveitamento, evitando retrabalho
* Devemos manter os padrões de templates.
* O Smartblock deve manter uma experiência clara e consistente para o usuário final

---

### Cenário de teste:

- [ ] Entrar na plataforma.
- [ ] Criar um Smartblock.
- [ ] Adicionar á uma regra.
- [ ] Entrar no site e fazer a jornada.

O Smartblock precisa realizar a trocar normalmente. de maneira instantânea.
