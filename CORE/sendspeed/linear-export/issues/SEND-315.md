## Descrição

**Descrição:** Atualmente o 'Webhook' aparece como opção disponível para o usuário. A mensagem "Configuração para `trigger.webhook` ainda não implementada. Em breve!" só aparece depois que o usuário já criou o card e clicou nele.

> **[Imagem 1 — transcrição]:** Screenshot (recorte pequeno) de um card de componente do tipo gatilho no Journey Builder: ícone verde de webhook, título "Webhook" e subtítulo "Inicia via chamada de API externa". Demonstra o componente "Webhook" que aparece como disponível na lista, mas cuja configuração ainda não está implementada.

**Sugestão de melhoria:** Exibir a mensagem "Configuração para `trigger.webhook` ainda não implementada. Em breve!" ANTES do usuário criar o componente, quando ele clicar no componente na lista de opções disponíveis. Assim ele já sabe que não está disponível antes de adicionar ao fluxo.
